## 🔍 How to Use

### 1️⃣ Browse Problems
Each problem is organized in its own directory with the LeetCode problem number:
```
📁 100-same-tree/
  ├── README.md          # Problem description & examples
  ├── solution.py        # Python solution
  ├── solution.js        # JavaScript solution
  └── notes.md           # Time/Space complexity & tips
```

### 2️⃣ View Solutions
```bash
# Clone the repository
git clone https://github.com/dipeshdarks/leetcode-dipdarks.git

# Navigate to a problem
cd 10-regular-expression-matching/

# Read the problem description
cat README.md

# Check the solution
cat solution.py
```

### 3️⃣ Run Locally
```bash
# Python
python solution.py

# JavaScript (Node.js)
node solution.js

# Java
javac Solution.java && java Solution
```

## 💡 Solution Approach

Each solution follows this structure:

### 📝 Problem Analysis
- **Input:** Problem constraints and examples
- **Output:** Expected return type and format
- **Constraints:** Size and value limits

### 🧠 Approach
```
1. Brute Force       → Understand the problem
2. Optimize          → Reduce time/space complexity
3. Refine            → Clean code implementation
4. Test              → Edge cases and validation
```
